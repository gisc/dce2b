Learning activities & branching logic

How the screens connect and how your accumulated state routes you to one of four endings. Purple = decision · teal = activity · blue = framing screen · red border = hard branch. Each node carries its Gagné event badge (teal G1–G9); the full mapping is below the diagram.

Legend

Decision point (DP) Learning activity (LA) Framing screen Hard / risk branch Outcome router G#Gagné event

Each DP shows a tradeoff layer when you pick a weak option — keep it (state stays “off”) or fix it. Activities award +10 and gate the Continue button.

Outcome router — evaluated top-to-bottom, first match wins

  1. Compounds == False → Ending C · Static Brain
  2. Secure == False AND Injection_Caught == False → Ending D · Compromised
  3. Uptime_OK || LowFriction || CostSustainable || Maintenance_On is False → Ending B · Abandoned
  4. otherwise → Ending A · Autonomous (success)

Gagné’s Nine Events of Instruction — how this course implements them

  1. G1Gain attention — animated hook + “your choices decide the ending.” → Start
  2. G2Inform learners of objectives — five concrete capabilities stated up front. → Objectives
  3. G3Stimulate recall of prior learning — “what happens when you close a chat?” → Warm-up
  4. G4Present the content — 60-second briefing on LLM Wiki + Hermes Agent. → Briefing
  5. G5Provide learning guidance — narration + coaching tradeoff layers. → every DP
  6. G6Elicit performance — make the six decisions; complete three activities. → DP1–DP6, LA1–LA3
  7. G7Provide feedback — tradeoff/risk layers, activity results, consequences. → layers & activity feedback
  8. G8Assess performance — running score, per-pillar scorecard, routed outcome. → Router & ending scorecards
  9. G9Enhance retention & transfer — build checklist, real resources, replay. → Apply-it screen
